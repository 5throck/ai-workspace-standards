# 프로젝트 문서

이 디렉토리에는 모든 프로젝트 문서와 아키텍처 아티팩트가 포함되어 있습니다.

## 파일

| 파일 | 목적 |
|------|------|
| `context.md` | 모든 AI 도구를 위한 단일 정보 출처 - 아키텍처, 기술 스택, 코딩 가이드라인, 멀티 에이전트 워크플로우 |
| `adr/` | 아키텍처 결정 기록 - 근거와 함께 중요한 기술적 결정들 |
| `security.md` | 보안 정책, 취약점 보고, 보안 개발 가이드라인 |

## 컨텍스트 파일 (`context.md`)

`context.md` 파일은 다음에 대한 주요 참고 문서입니다:

- **프로젝트 개요**: 기술 스택, 아키텍처, 폴더 구조
- **개발 워크플로우**: Git 규칙, PR 프로세스, 동기화 파이프라인
- **코딩 가이드라인**: 언어 규칙, 파일 인코딩, 오픈소스 정책
- **멀티 에이전트 시스템**: 에이전트 역할, 스킬, 디스패치 프로토콜
- **주요 파일**: 모든 중요한 프로젝트 파일 참조

모든 AI 도구(Claude Code, Gemini CLI, Codex)가 세션 시작 시 이 파일을 로드합니다.

## 아키텍처 결정 기록 (ADR)

ADR은 다음 내용으로 중요한 아키텍처 결정을 문서화합니다:

- **컨텍스트**: 결정이 필요한 이유
- **결정**: 무엇을 결정했는지
- **영향**: 긍정적 및 부정적 영향
- **고려된 대안**: 평가된 다른 옵션

### ADR 생성

1. `_examples/adr/0001-example-decision.md` 템플릿 복사
2. 순차적 번호로 이름 변경: `adr/NNNN-title.md`
3. 결정 세부 정보 작성

### ADR 수명 주기

| 상태 | 의미 |
|------|------|
| Proposed | 검토 중 |
| Accepted | 현재 접근 방식 |
| Deprecated | 더 이상 권장되지 않음 |
| Superseded | 최신 ADR으로 대체됨 |
