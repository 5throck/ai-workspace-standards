### 6.5 Script Lifecycle Management {#script-lifecycle-management}

Scripts are managed across three ownership layers. Development originates at L0 (workspace) and is explicitly published to L1 (template) via `publish-to-template.sh`.

#### Ownership Layers

| Layer | Location | Owner | Update Policy |
|-------|----------|-------|---------------|
| **L0 â€” Workspace SSOT** | `scripts/` (workspace root) | workspace maintainer | Versioned via `scripts/SCRIPTS.md` |
| **L1 â€” Template snapshot** | `templates/common/scripts/` | publish: `bash scripts/publish-to-template.sh` | Explicit publish from L0 |
| **L2 â€” Project** | `<project>/scripts/` | project team | Independent snapshot after creation |

**Propagation rule**: Development happens at L0 (workspace `scripts/`). Publish L0â†’L1 explicitly using `bash scripts/publish-to-template.sh` before creating new projects. L2 is a snapshot of L1 at project creation time â€” no automatic back-propagation from L2.

#### Script States

| Status | Meaning | Enforcement |
|--------|---------|-------------|
| `active` | In production use | Version bump required on change |
| `deprecated` | Scheduled for removal | `removal-date` required (90-day minimum notice); `dev-sync` warns L1/L2 |
| `experimental` | Unstable / not propagated | Not synced to L1/L2 automatically |

#### Dependency Tracking

Scripts that call other scripts must declare dependencies in `SCRIPTS.md` Registry:

The `verify-scripts.ts` script checks for:
- âś… Circular dependencies (A depends on B, B depends on A)
- âś… Missing dependencies (declared dependency doesn't exist)
- âś… Stale dependencies (declared dependency has been deprecated)

#### Deprecation Protocol

1. Set `status: deprecated` and `removal-date: YYYY-MM-DD` in `SCRIPTS.md` Registry (â‰Ą 90 days notice)
2. `dev-sync.sh` warns all L1/L2 consumers on every run until removed
3. On `removal-date`, `verify-scripts.ts --verify` **hard blocks** pre-commit

#### Security Advisory Protocol

When a security vulnerability is found in a script:
1. Set `security-advisory: CVE-XXXX` in `SCRIPTS.md` Registry
2. `dev-sync.sh` **hard blocks immediately** in L1/L2 (no grace period)
3. Advisory is cleared only after the script is patched or removed
