# Migration Guide: co-security 0.1.0 → 0.2.0

**Release Date:** 2026-05-28  
**Mandatory:** Yes (security update)  
**Template:** co-security  
**Status:** Beta

## Breaking Changes
None

## New Features
- verify-authorization skill with signature date validation
- PostToolUse hook enabled by default for audit compliance
- Enhanced authorization.md with signature tracking
- Security engagement workflow with mandatory verification steps

## Migration Steps

### 1. Review New Security Engagement Rules
Read the updated security engagement documentation:
```bash
cat docs/security-engagement.md
```

### 2. Update Authorization Documentation
Update your `docs/authorization.md` with new required fields:
- signature_date
- engagement_scope
- security_clearance_level

Example:
```markdown
# Authorization

**Project:** [PROJECT_NAME]
**Authorized By:** [AUTHORIZER]
**Signature Date:** 2026-05-28
**Engagement Scope:** [SCOPE_DESCRIPTION]
**Security Clearance Level:** [LEVEL]
```

### 3. Run Verification
Verify your authorization is properly configured:
```bash
/skill verify-authorization
```

### 4. Test PostToolUse Hook
Test that the audit hook fires correctly:
```bash
bash scripts/audit.sh
```

## Rollback Guide
If issues occur after migration:

1. Revert to previous template version:
```bash
git checkout <previous-commit-hash>
```

2. Restore previous authorization format:
```bash
git checkout HEAD~1 docs/authorization.md
```

3. Disable PostToolUse hook temporarily (if needed):
Edit `.claude/settings.json` and remove the PostToolUse hook configuration.
