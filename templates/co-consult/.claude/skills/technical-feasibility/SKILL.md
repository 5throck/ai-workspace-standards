## Overview

This skill provides a structured framework for assessing whether proposed solutions are technically feasible given client constraints, existing infrastructure, and required integrations.

## When to Use This Skill

- Trigger: "Technical feasibility" or "Can we technically do this?"
- Use Case: Validating solution design assumptions and surfacing technical risks
- Phase: Phase 2 (Solution Design), Phase 3 (Deliverable Creation)

## Steps

### Step 1: Define Technical Scope
- Identify the technology components in the proposed solution
- Document current-state technology landscape
- List integration and dependency requirements

### Step 2: Assess Technical Fit
- Evaluate alignment with existing technology standards
- Identify capability gaps requiring new technology or skills
- Assess vendor maturity and market availability

### Step 3: Identify Technical Risks
- Classify risks by severity (high/medium/low)
- Assess integration complexity and data migration needs
- Evaluate security and compliance implications

### Step 4: Feasibility Recommendation
- Rate overall technical feasibility (high/medium/low)
- List pre-conditions for technical implementation
- Recommend mitigation actions for key risks

## Expected Outputs

- Technical feasibility assessment
- Risk register (technical dimension)
- Technology requirements list
