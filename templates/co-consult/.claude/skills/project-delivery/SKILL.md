## Overview

This skill provides structured tools for managing consulting engagement delivery, ensuring projects are delivered on time, within scope, and with quality outcomes.

## When to Use This Skill

- Trigger: "Project plan" or "Delivery management" or "Workstream status"
- Use Case: Managing Phase 4 implementation and delivery execution
- Phase: Phase 4 (Implementation & Delivery)

## Steps

### Step 1: Define Project Structure
- Break engagement into workstreams and milestones
- Assign ownership per workstream
- Establish governance cadence (steering, working, status)

### Step 2: Build Delivery Plan
- Create detailed task list with owners and due dates
- Identify critical path and dependencies
- Define milestones and deliverable acceptance criteria

### Step 3: Execute and Track
- Monitor progress against plan weekly
- Identify and escalate risks and issues early
- Manage scope changes through formal change control

### Step 4: Report and Close
- Produce weekly status reports for client and internal leadership
- Conduct milestone reviews against acceptance criteria
- Document lessons learned and transition to operations

## Expected Outputs

- Project plan (workstreams, tasks, milestones)
- Weekly status report template
- Risk and issue log
