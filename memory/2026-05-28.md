## Decisions
- **Platform Profile default = `both`**: new-project and upgrade-project default to including both CLAUDE.md and GEMINI.md. Users must explicitly pass `--platform claude` or `--platform antigravity` to exclude one.
- **WORKSPACE-MANAGED marker as upgrade boundary**: MERGE-tier files use HTML comment markers to delimit template-owned vs. user-owned sections; upgrade scripts replace only marked regions, never touching user content outside markers. Defined in CONSTITUTION.md §10.
- **Security Bootstrap as hard stop (not warning)**: any of the 5 security checks failing aborts scaffolding/upgrade with `exit 1`. Soft warnings were explicitly rejected — a failed hook setup is a blocking defect.
- **git stash before file writes**: the pre-upgrade snapshot must precede all LOCKED/MERGE file overwrites to enable true rollback via `git stash pop`. Bug where stash ran after writes was caught and fixed by Auditor QA gate.
- **docs-writer tier Medium**: role split confirmed — Architect owns document architecture decisions (section structure, inter-file relationships); DocsWriter executes (writing, editing, terminology consistency, translation). Medium tier required for terminology governance tasks.
- **CONSTITUTION.md §10 as canonical terminology source**: all future agents and scripts must use the exact terms defined there (Template Variant, Platform Profile, etc.) — no synonyms.
- **2 multi-agent meetings held** (team composition + implementation direction) before code changes began; all 5 action items derived directly from meeting synthesis.

## Open Issues
- `upgrade-project.sh` MERGE logic uses python3 for regex replacement — requires python3 on PATH; no fallback for environments without python3. Consider pure bash sed/awk alternative.
- `templates/minimal/` bare template (for pure code projects without AI agents) deferred — out of scope for this session, flagged for future quarterly template sync.
- `agents/*.md` MERGE section boundary in upgrade-project: no WORKSPACE-MANAGED markers exist in current agent files, so upgrade treats them as "no markers → append". Architect needs to define which sections of agent files are workspace-managed vs. project-extensible (separate ADR).
- `validate-templates.ts` Script Parity Annotation check not yet implemented — CONSTITUTION.md §10 defines the `# [parity:<tag>]` annotation format but the validator has not been extended yet (A-03 from first meeting).
