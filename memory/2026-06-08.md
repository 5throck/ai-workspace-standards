# Decision Record: Fix Variant Scaffolding Validation Failure

- **Date**: 2026-06-08
- **Topic**: Fix variant scaffolding validation failure
- **Status**: Approved

## Context
Creating new projects with `new-project.sh` failed in the "Validating template integrity..." phase due to missing `CLAUDE.md` and `GEMINI.md` in variant template directories (e.g., templates/co-develop, templates/co-design).

## Analysis
Individual variant templates do not contain `CLAUDE.md` or `GEMINI.md` at their root by design. Instead, they inherit them from `templates/common` at project creation time, and only override them if required. `template-validation.ts` mistakenly listed them as mandatory for all variants.

## Decision
Removed `CLAUDE.md` and `GEMINI.md` from `variantRequired` in `scripts/helpers/template-validation.ts`, and bumped its version from `1.0.0` to `1.0.1`. Update version registry accordingly.

# Decision Record: Fix extends path security validation and documentation false positives during new project scaffolding

- **Date**: 2026-06-08
- **Topic**: Fix extends path security validation and documentation false positives during new project scaffolding
- **Status**: Approved

## Context
Creating new projects with `new-project.sh` failed in the post-scaffold audit phase. The extends validation failed for `co-develop/agents/pm.md` because `extends: ../../common/agents/pm.md` was resolved relative to the copied project directory rather than the template source directory. Additionally, the extends verification flagged an unresolved extends field in `docs/variant-pm-spec.md` due to an unindented example code block.

## Decision
1. Updated `merge-frontmatter.ts` (v1.1.1) to support a 3rd CLI argument `original-context-path`. When provided, the validator validates this path instead of the copied target path.
2. Updated `new-project.sh` (v1.4.7) to pass the template source path (`$src_file`) as the 3rd argument to `merge-frontmatter.ts`.
3. Indented the code block in `templates/common/docs/variant-pm-spec.md` to avoid matching `^extends:`.
