# Session Log — 2026-07-08

## Variant Registry Architecture + co-game Variant Creation

### Overview

Completed a two-phase initiative:
- **Phase A**: Infrastructure refactoring — centralized Variant Type Registry, Capability Registry, Plugin System, Validator Framework, and Transactional Workspace Integration
- **Phase B**: co-game variant creation from `Projects/co-game` → `templates/co-game/`

### Phase A — Infrastructure Refactoring

#### 1. Centralized Registries (`scripts/helpers/registries/`)
- **variant-type-registry.ts**: SSOT for 7 variant types (security, development, design, consulting, collaboration, lecture, game). Uses `as const` + `keyof typeof` pattern.
- **capability-registry.ts**: 21 capabilities with `isCapability()` runtime guard. Const object (not enum).
- **promotion-policy.ts**: Per-variant-type promotion rules. Game: minEngagements=3, minBetaMonths=3.
- **validation-policy.ts**: Per-variant-type validation rules. Game: required capabilities + agents.
- **index.ts**: Barrel exports + `validateRegistryIntegrity()` with 7 cross-registry checks.

#### 2. Plugin System (`scripts/helpers/plugins/`)
- **variant-plugin.ts**: `VariantPlugin` interface with 9 lifecycle hooks. Internal plugin map with explicit registration.
- **game-plugin.ts**: `GamePlugin` implementing capability coverage + performance budget + architecture pattern checks.
- **index.ts**: Explicit `registerPlugin(new GamePlugin())` — NOT self-register on import.

#### 3. Validator Framework (`scripts/validators/`)
- **types.ts**: ValidatorContext, ValidationIssue, ValidatorResult, ValidationReport types.
- **6 validators**: variant-json, extends-validator-wrapper, capability, orphan-reference, duplicate, platform-parity.
- **index.ts**: `runAllValidators()` with dependency ordering + `generateReport()` for CI-friendly JSON.
- Each validator has `prerequisites[]` field — skip if prerequisite failed.

#### 4. Workspace Integration (`scripts/helpers/workspace-integration.ts`)
- 5-phase transaction: Preflight → Begin (snapshot) → Write → Verify → Commit/Rollback.
- Auto-updates: propagation-map.json, VERSION_REGISTRY.json, README.md, new-project scripts, AGENTS.md.
- Fixes VERSION_REGISTRY schema mismatch (writes nested object format, not array).

#### 5. Pipeline Enhancements (`scripts/l2-to-variant-pipeline.ts`)
- Phase 3.7: Plugin validation hook (game-plugin capability checks).
- Phase 4.5: Enhanced JSON report format with category, status, duration_ms, validator, severity, checks.
- Phase 7: Deprecated (defaults OFF) — delegated to workspace-integration.ts.

#### 6. Consumer Refactoring
Refactored existing scripts to import from centralized registries:
- `variant-governance-rules.ts` — removed local VariantPromotionCriteria union
- `golden-reference-loader.ts` — removed local VariantType union + LAYER2_*_SECTIONS
- `beta-lifecycle.ts` — removed string literal unions
- `generate-variant.ts` — removed VariantMetadata.variantType union + 3 inline functions

#### 7. Bug Fix
- `scripts/lib/pipeline-state.ts`: Fixed `clearState()` — was missing `async` keyword despite using `await`. Version bumped to 1.1.1.

### Key Decisions
- Policy vs Plugin boundary: Policy = declarative rules (data-driven), Plugin = procedural rules (code-driven)
- No Zod dependency: all validation is hand-rolled TypeScript (matches existing codebase pattern)
- No `createdAt` in registry: git tracks history
- Plugin explicit registration: NOT self-register on import
- Phase 7 deprecated in pipeline: delegated to workspace-integration.ts
