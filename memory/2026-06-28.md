## Decisions
- Narration (TTS) and auto-advance are two independent features with separate config sections in lecture-profile.md
- Each feature has its own `enabled` flag controlling UI visibility independently
- `auto_advance.start_as_auto` replaces the old `narration.autoAdvance` (which was "informational only" and ignored by runtime)
- New `narration.auto_play` field for auto-starting TTS on page load
- snake_case naming for all new fields (consistent with `background_image`, `source_verification`)
- Backward compatible: old profiles without `auto_advance:` section default to enabled/manual; old `narration.autoAdvance` silently ignored
