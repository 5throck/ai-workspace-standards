## Session Summary
investigate(co-deck): auto-advance not working in some HTML builds

## Changes
- `CHANGELOG.md` — reverted incorrect changelog entry
- `templates/co-deck/docs/html-themes/themes/_shared/ppt-engine.js` — reverted incorrect showSlide() guard change

## Decisions
- Initial hypothesis (showSlide() guard gating _onSlideChanged on isPlaying) was WRONG. The template engine code is correct — v2 HTML proves auto-advance works with the current engine.
- **Actual root causes identified by comparing 3 HTML files:**
  1. **korean-food v1**: Built with pre-v2.1 NarrationEngine — `_startAutoAdvanceTimer`, `_stopAutoAdvanceTimer`, `_restartAutoAdvanceTimer` functions entirely missing. `toggleAutoAdvance()` toggles state but never starts timer. This is a build artifact from an older html-build run that inlined an outdated engine.
  2. **v5 (ai-marketing)**: Engine code is correct (v2.1 with all timer functions), but `autoAdvanceInterval: 1` (1 second) in narrationConfig. At 1s, timer fires every second but is suppressed by `isPlaying` guard during narration; causes confusing behavior when narration is off (slides advance too fast) or on (double-advance race with speech-end handler).
  3. **v2 (ai-marketing)**: Works correctly — same engine code as v5 but with `autoAdvanceInterval: 8`.
- **No template fix needed** — the ppt-engine.js source template is correct. The issue is in the build output (stale engine inlined in korean-food v1) and config injection (wrong interval in v5).
- To fix affected presentations: re-run html-build to re-inline the latest ppt-engine.js into the HTML files.

## Open Issues
- korean-food v1 needs re-build with current engine (stale NarrationEngine inlined)
- v5 needs narrationConfig.autoAdvanceInterval corrected from 1 to 8
